## 一、KPAX 是什么

**你的人生，是你选择的总和。**

KPAX 是一个多专家 AI 辩论分析工具。用户输入一个复杂问题，KPAX 自动召集 3-5 个 AI 专家进行结构化辩论，最终输出一份多角度分析报告。全程不超过 3 分钟。

名字致敬电影 K-PAX (2001)——Prot 这个角色，没人知道他说的是不是真的，但他看问题的角度比在场所有人都深刻。KPAX 做的事情一样：给你一个你自己没想到过的角度。

### 用户场景

- "我该不该现在买房？"
- "这个创业方向靠谱吗？"
- "35岁该不该转行做AI？"
- "世界杯这场比赛怎么看？"

这些问题的共同点是：没有标准答案，但你可以做更充分的分析再做决定。KPAX 就是帮你把分析这一步做到位。

### 核心流程

```
用户一句话输入
    ↓
AI 自动识别问题涉及的领域，生成对应的专家阵容
    ↓
3-5 个 AI 专家进行 2-3 轮结构化辩论
    ↓
输出一份一页纸分析报告：
  - 问题的多角度拆解
  - 支持论点和风险论点（都附依据）
  - 专家之间的分歧点
  - 如果要深入研究的方向建议
  - 相关学术研究引用（来自 OpenAlex）
```

### 商业模式

订阅制为主（$5-15/月），按次付费为辅。用户为"更好的判断依据"付费，本质上是知识付费。法律合规上，KPAX 是分析工具，不碰资金，不做投注。

## 二、为什么做

### 两个产品，一套引擎

KPAX 不是凭空冒出来的产品。它脱胎于 Agent X Lab——一个我们已经做了相当长时间的开源学术跨学科研究平台。

| | Agent X Lab | KPAX |
|---|-------------|------|
| 定位 | 开源学术跨学科研究平台 | 多专家 AI 辩论分析工具 |
| 目标 | GitHub stars + 学术影响力 | 用户增长 + 商业收入 |
| 受众 | 研究生、学者、开发者 | 任何面对复杂决策的人 |
| 商业模式 | 免费开源 | 订阅 / 按次付费 |

两者的关系不只是"共用代码"这么简单。Agent X Lab 是 KPAX 的研发引擎——我们用它的交叉学科发现能力来指导 KPAX 每个场景的专家阵容应该怎么设计，而不是拍脑袋决定。后面世界杯场景的专家阵容就是这么来的。

### 市场现状

多 Agent 辩论这个赛道已经有人在做了（71vote、arbitrIQ、Verdikt、Suprmind），但没有跑出来的赢家。原因是两极分化严重：

```
轻量但没深度                     有深度但很贵
71vote ─────────── [KPAX] ─────────── arbitrIQ
(免费/纯角色扮演)    (中间地带)        (49欧/次/面向高管)
```

71vote 本质上是角色扮演，没有真正的知识支撑；arbitrIQ 做得认真但定价太高，普通人用不起。中间这个位置——有深度、用得起、一句话就能开始——目前没有人占。

### 竞品对比

| | KPAX | 71vote | arbitrIQ | ChatGPT |
|---|------|--------|----------|---------|
| 分析深度 | 有学术文献支撑 | 纯角色扮演 | 有 web search | 单一视角 |
| 价格 | $5-15/月 | 免费 | 49欧/次 | $20/月 |
| 使用门槛 | 一句话输入 | 一句话输入 | 需上传文件 | 一句话输入 |
| 输出形式 | 结构化报告 | 讨论文字 | 决策报告 | 一段回答 |
| 记忆积累 | 越用越强 | 无 | 无 | 有限 |

### 关于护城河

说实话，现阶段没有技术护城河。LLM API 加 prompt 谁都能做，OpenAlex 数据也是公开的。现阶段拼的是速度和执行力。但护城河会在增长过程中自然形成：用户数据积累、Agent 分析的历史表现、记忆系统的飞轮效应、以及社区。这些东西别人没法用 Claude Code 三天复制出来。

## 三、技术架构

### 原则：不重复造轮子

和做 Agent X Lab 时的做法一样——能用现成开源方案的直接封装进来，自己只做别人没有的部分。

### 整体结构

```
用户层（KPAX 前端）
    ↓
编排层（Claude / OpenAI Assistants API — 用现成的）
    ↓  ↑
┌───────────────────────────────────────────┐
│         KPAX 自有核心                      │
│  ┌─────────────┐  ┌────────────────────┐  │
│  │ 记忆系统     │  │ 领域专家 + 报告生成 │  │
│  │ (封装Hermes) │  │ (世界杯等场景优化)  │  │
│  └──────┬──────┘  └────────┬───────────┘  │
│         │                  │               │
│  ┌──────┴──────────────────┴──────────┐   │
│  │  Agent X Lab 引擎层                 │   │
│  │  反向发现 + 辩论引擎 + 知识图谱     │   │
│  └────────────────────────────────────┘   │
└───────────────────────────────────────────┘
```

**用别人的**：Agent 编排（Claude/OpenAI API）、LLM 推理（LiteLLM）、记忆架构设计模式（Hermes，MIT 协议）

**自己做的**：记忆数据的积累和管理、领域专家 prompt 优化、报告生成器、Agent X Lab 的辩论引擎和知识图谱

### 从 Agent X Lab 复用

这些模块已经在 Agent X Lab 1.0 里跑通了，直接搬过来：

- 多 Agent 辩论引擎（debate_engine.py）
- AI provider 层（LiteLLM 多模型调度）
- 知识图谱数据层（OpenAlex 4516 个学术 topic）
- 反向发现引擎（从一个问题自动识别涉及哪些学科）
- Token 配额系统

### 记忆系统

这是 KPAX "越用越强"的核心机制，也是长期壁垒所在。架构上借鉴 Hermes Agent（NousResearch 开源，MIT 协议，57k+ stars），这是目前开源社区最成熟的 agent 记忆方案。我们不 fork 它的代码，而是封装它的设计模式。

四层记忆，各层解决不同的问题：

| 层级 | 设计来源 | 解决什么问题 | 在 KPAX 里的具体应用 |
|------|---------|-------------|---------------------|
| L1 持久记忆 | Hermes MEMORY.md + USER.md | 用户是谁、关心什么 | 用户的投注偏好、关注联赛、风险偏好。会话开始时注入，中间不改动（冻结快照模式，省 token 且稳定）。接近容量上限时自动压缩合并。 |
| L2 技能记忆 | Hermes Skills 系统 | 避免重复劳动 | 跑完一次"小组赛出线压力局"分析后，自动沉淀为可复用的分析模板。下次遇到类似情境直接加载，不从头开始。渐进式加载：先加载模板名，按需加载内容。用户越多，模板库越丰富。 |
| L3 会话搜索 | Hermes SQLite FTS5 + Zep Cloud | 找到历史上的类似分析 | "有人分析过巴西 vs 德国吗？"——全文检索 + 语义搜索，找到相关的历史分析作为参考或直接复用。 |
| L4 共享记忆 | Hermes PLUR 插件 | 跨用户知识积累 | 用户 A 分析了巴西的防守数据，用户 B 问巴西 vs 阿根廷时自动继承这些经验。赛后验证某次分析有误时，修正自动传播到所有相关记忆。这就是飞轮：用户越多，积累越多，分析越准。 |

### 需要新做的模块

- 通用专家角色生成器（不局限于学科学者，能生成经济学家、律师、心理学家等角色）
- 问题解析器（一句话 → 识别涉及哪些领域 → 自动配置专家阵容）
- 报告生成器（把辩论结果整理成结构化的一页纸报告）
- 极简前端（核心就是一个输入框）
- 记忆封装层（把 Hermes 的设计模式和 Zep Cloud 对接起来）

## 四、第一个场景：2026 世界杯

### 为什么选这个

2026 世界杯即将到来，时间窗口非常明确。这也是创始人自己最想用的场景——"这场球怎么看"。体育分析用户的付费意愿很强，而且天然高频（每天都有比赛）。单场比赛分析是最小的可验证单元，适合作为 MVP 的第一个切入点。

### Agent X Lab 的验证

我们用 Agent X Lab 自己的交叉学科引擎跑了一次"如何科学预测足球比赛结果"的分析。概率统计、高级统计方法、复杂网络分析三个学科的专家辩论后，形成了一个共识——科学地预测足球需要三层能力：

1. **动态建模** — 追踪球队战术状态随时间的变化，不是看静态数据快照
2. **网络分析** — 用图网络方法分析传球网络的结构特征，包括配合效率、网络密度、协同模式
3. **统计预测** — 用贝叶斯模型把前两层的数据融合在一起，输出带置信度的预测结果

这个结论直接指导了下面专家阵容的设计——不是我们拍脑袋觉得应该有什么专家，而是学术辩论告诉我们这个问题需要什么专业能力。

### 诚实面对预测的天花板

足球是所有主流运动里最难预测的。一场比赛只有 0-3 个进球，一个门柱、一个误判就能改变结果，信号噪声比极低。同一对阵组合的历史数据极少（世界杯一支球队才踢7场），球员心理状态、临时变阵、伤病真实影响这些关键变量几乎无法量化。

目前的天花板：预测市场（Polymarket、Betfair）大约 60-70%，学术模型 55-65%，顶级量化博彩团队 52-55%。没有任何人或模型能稳定超过 70%。

但对于经常做决策的用户来说，哪怕只提高 5 个百分点的判断质量，长期下来就是盈亏线的差距。

### 专家阵容（基于 Agent X Lab 的学术发现设计）

| 专家角色 | 负责的分析维度 |
|---------|---------------|
| 战术网络分析师 | 传球网络的效率变化、攻防结构模式的动态演变 |
| 统计建模专家 | 贝叶斯预测模型的输出、不确定性量化、置信度计算 |
| 战术解读专家 | 把模型和数据的输出翻译成普通人能理解的战术分析 |
| 心理情境分析师 | 球队士气、大赛压力、主客场心理、历史交锋的心理影响 |

### 单场报告模板（草案）

```
[比赛] A vs B | [赛事] 世界杯小组赛

## 核心判断
A 胜率: xx% | 平局: xx% | B 胜率: xx%
置信度: [高/中/低]（附原因）

## 五维分析
1. 战术网络 — 近5场传球网络效率趋势
2. 统计模型 — ELO/贝叶斯模型输出
3. 阵容伤病 — 关键球员状态评估
4. 心理情境 — 出线压力、大赛经验、历史交锋
5. 赔率市场 — 当前市场定价 vs 我们分析的偏差

## 关键变量
- 如果 [X] 发生，判断会翻转
- 最大不确定性来源：[具体指出]

## 专家分歧
- [专家A] 和 [专家B] 在哪个问题上看法不同
- 没有达成共识的点
```

## 五、落地计划

### Phase 1：验证核心体验

**目标**：10 个人用完之后，至少 3 个愿意用第二次。

**做**：
- 一个页面：输入框 + 报告展示
- 后端：问题解析 + 专家生成 + 2轮辩论 + 报告输出
- 部署到一个可访问的地址

**Phase 2+（验证通过后再加）**：
- 支付接入（验证通过后立即做）
- 预测市场 API 集成（Polymarket / Betfair 数据接入）
- 社区功能
- 移动端
