# Agent X Lab — 系统架构

> "今天想发现点什么？" — 让学术变得有趣的交叉学科发现平台

## 产品定位

Agent X Lab 是一个交叉学科研究的**发现与协作平台**。AI Agent 加速科研的"前半段"（发现空白 + 理论论证），社区论坛承接"后半段"（实验验证 + 成果反馈），形成完整的科研加速闭环。

平台不替代科研，而是把"值得做的方向"高效地送到"有能力做的人"手里。

**使命：降低跨学科研究的门槛，让全世界的研究者都能参与到原本被信息壁垒阻隔的科学发现中。**

## 产品哲学：AI 做理论，社区做实验

> 这是 Agent X Lab 最核心的设计理念，所有功能都围绕这条链路展开。

### 科研的两半

传统科研流程中，"发现方向"和"验证方向"是一个人或一个团队从头做到尾的。但这两半的瓶颈完全不同：

- **前半段（发现+理论）** — 瓶颈是信息壁垒和学科视野局限。一个心理学教授很难知道 AI 领域的最新进展能如何应用到他的研究中。这正是 AI 擅长的：跨领域关联、快速综合、不受学科边界限制。
- **后半段（实验+验证）** — 瓶颈是物理资源：实验室、被试、数据集、IRB 批准、仪器设备。这是 AI 做不了的，必须由真实的研究者来做。

Agent X Lab 的定位是：**用 AI 解决前半段，用社区解决后半段。**

### 完整闭环

```
发现空白        理论论证        社区接力        知识回馈
   │               │               │               │
   ▼               ▼               ▼               ▼
用户在画布    →  Agent 辩论   →  论坛帖自动    →  实验结果
发现跨学科       论证可行性      生成，标注       贴回论坛
空白（虚线）     产出研究方向    "缺实验验证"     │
                 生成论文草稿                     ▼
                                              知识图谱更新
                                              虚线 → 实线
                                              新的 paper 入库
```

### 为什么不让 AI Agent 模拟实验？

1. **真实性** — AI 模拟的实验数据没有科学价值。真正的实验需要真实的被试、真实的设备、真实的伦理审查。
2. **资源分配** — 世界各地的实验室有不同的资源优势。一个有 fMRI 的神经科学实验室和一个有大规模用户数据的科技公司，能做的实验完全不同。论坛让方向和资源自然匹配。
3. **社区价值** — 如果 AI 从头做到尾，研究者就没有参与感。让社区承接实验环节，研究者才会真正"拥有"这个方向，愿意投入资源和声誉。
4. **知识积累** — 社区的实验结果反哺到知识图谱，让 AI 下一轮辩论时有更扎实的证据基础。这是一个越用越强的正循环。

### AI Agent 在实验维度的正确角色

AI Agent 不做实验，但可以做**实验方案设计建议**：

- "这个假设适合用 A/B 测试还是纵向追踪？"
- "统计功效分析建议样本量至少 N=120"
- "这个跨学科设计需要注意的混杂变量有..."
- "类似的实验设计参考：[论文 A], [论文 B]"

这就是 Future 规划中 experimentalist role 的定位——不替代实验，而是帮社区里接手实验的人设计更好的方案。

## 设计原则

- **对话优先，图谱生长**: 入口是搜索/对话，不是全量图谱。用户选几个学科放几个节点，图谱从空白逐步生长
- **已有联系 = 实线，未知交叉 = 虚线闪烁**: 没有现有研究的交叉 = 最好的研究机会
- **AI 做理论，社区做实验**: 平台不替代完整科研流程，而是加速前半段（发现+理论），把后半段（实验+验证）交给社区中有资源的人
- **学术但有趣**: 圆角卡片、明亮配色、微动画，Duolingo 的温暖感 + 学术的专业底色
- **侧边栏 2 级展示**: Field（26 个）+ 展开 Subfield（每组约 10 个），Topic 交给搜索/AI

## 信息源架构

> 核心原则：Agent X Lab 不是"全学科数据库聚合器"，而是"跨学科路由器"。
> 专业性来自正确的分层和标注，不是来自接口数量。

### 三层信息源模型

```
┌─────────────────────────────────────────────────────┐
│  第一层: 探索层 (自由发散，允许不够权威但有启发)        │
│  ── 概念发现、跨学科跳转、关键词扩展、作者关系、引用链   │
│                                                       │
│  已接入: OpenAlex (works/authors/topics 关系图谱)      │
│  已接入: Zep Cloud (Agent 记忆 + 知识积累)             │
│  规划中: Crossref (DOI/元数据/引用关系)                │
│  规划中: Unpaywall (开放获取全文判断)                   │
│  规划中: Wikipedia (概念导航/术语扩展)                  │
├─────────────────────────────────────────────────────┤
│  第二层: 证据层 (收紧，只保留可作为研究依据的内容)       │
│  ── 正式期刊论文、综述、meta-analysis、机构报告         │
│                                                       │
│  规划中: arXiv (CS/物理/数学/统计 预印本)              │
│  规划中: SSRN (经济/金融/法学/管理 working paper)      │
│  规划中: PubMed/Europe PMC (生命科学权威索引+全文)      │
│  规划中: ERIC (教育研究)                               │
├─────────────────────────────────────────────────────┤
│  第三层: 裁决层 (按需进入，高权威专业验证)               │
│  ── 用户主动请求"严谨验证"时才触发                     │
│                                                       │
│  按学科按需接入:                                       │
│    医学 → PubMed / PMC / PubChem                      │
│    CS   → DBLP / arXiv / IEEE Xplore(付费)            │
│    经济 → RePEc / EconLit(付费)                       │
│    心理 → APA PsycInfo(付费)                          │
│    教育 → ERIC                                        │
│    地学 → GeoRef(付费)                                │
│    法学 → HeinOnline(付费)                            │
│    人文 → MLA Bibliography / Project MUSE / JSTOR(付费)│
│    化学 → PubChem + 专业化学数据库(付费)               │
│    农业 → AGRICOLA                                    │
└─────────────────────────────────────────────────────┘
```

### 结果标注系统 (Evidence Tagging)

每条检索结果必须携带以下六维标签，直接影响排序和展示：

| 维度 | 取值示例 | 用途 |
|------|----------|------|
| 来源性质 | 百科 / 预印本 / 期刊论文 / 综述 / 官方报告 / 数据库记录 | 区分内容类型 |
| 证据等级 | 启发 / 参考 / 证据 / 权威 | 排序权重 |
| 学科归属 | 医学 / 心理学 / CS / 经济学 / ... | 路由判断 |
| 评审状态 | 同行评审 / 预印本 / 编辑性内容 / 未知 | 可信度判断 |
| 可追溯性 | 有 DOI / 有 PMID / 有正式出版信息 / 无正式标识 | 引用可靠性 |
| 全文状态 | 开放获取 / 仅摘要 / 需订阅 / 未知 | 可读性判断 |

### 学科路由器 (Discipline Router)

系统不是"一个搜索框搜全库"，而是：

1. **判断问题类型** — LLM 分析用户输入，识别涉及哪些学科
2. **选择信息源** — 按学科匹配将请求路由到对应数据源
3. **分级展示** — 探索层结果和证据层结果分区展示，不混排
4. **风险提示** — 高风险问题（医疗/法律/金融）自动标注"探索性结果，不构成专业结论"

### 成本策略

| 类别 | 数据源 | 成本 | 阶段 |
|------|--------|------|------|
| 零成本骨架 | Crossref + OpenAlex(免费额度) + Unpaywall + PubMed/PMC + ERIC | 免费或接近免费 | MVP |
| 低成本增强 | OpenAlex 高配额 API + arXiv + SSRN + DBLP + RePEc | API 用量费 | Phase 2 |
| 商业增强 | Web of Science + Scopus + JSTOR + IEEE Xplore + HeinOnline | 机构订阅/销售报价 | 按需 |

## 数据模型

### OpenAlex 学科层级（映射到 Discipline 表）

```
Domain (4)  →  不在界面展示
  └── Field (26)  →  侧边栏第 1 级
       └── Subfield (254)  →  侧边栏第 2 级（点击展开）
            └── Topic (4,516)  →  搜索/AI 推荐使用，不在侧边栏
```

### 核心实体

- **Discipline**: 学科节点（self-referencing tree: Field → Subfield → Topic）
- **Scholar**: 学者
- **Paper**: 论文
- **Intersection**: 两个学科的交叉关系（多对多超边）
- **AIHypothesis**: AI 生成的研究假说
- **Debate**: 辩论会话（模式: free/debate，关联 intersection，四段式总结）
- **DebateAgent**: 辩论参与 Agent（学科身份 + 职级 rank + 权重 weight + 性格 persona + 立场 stance + system prompt）
- **DebateMessage**: 辩论消息（Agent 发言，按轮次编号）
- **PaperDraft**: 论文草稿（关联 Debate，含标题/方向/状态）
- **PaperSection**: 论文章节（排序/标题/描述/写作指令/内容/版本号/状态）
- **ForumPost**: 论坛帖子
- **ForumComment**: 论坛评论
- **User**: 用户（积分、偏好）
- **DiscoveryPoint**: 发现点积分记录

## 页面结构

### 1. 首页

- "今天想发现点什么？" 搜索/对话入口
- 热门学科标签（快速选择 Field）
- "大家在研究什么" 热门方向卡片
- 论文排行榜

### 2. 画布页

- 顶部: AI 发现搜索栏（输入研究问题 → 调用 /api/discover → 自动匹配学科并高亮）
- 左侧: 学科树（Field + Subfield 展开，点击 toggle 选中/取消）
- 中央: D3 力导图
  - **起始空白**：不传 IDs 时后端返回空图，画布显示引导文字
  - **动态生长**：用户每选一个学科，前端带 `?ids=` 参数调 `/api/graph`，只返回被选学科作为节点
  - **交叉连线**：如果被选学科之间已有 Intersection 记录，自动出现连线
  - **研究空白**：选 2+ 学科但无交叉 → 提示 "研究空白" → 可触发 AI 假设生成
- 右侧: 详情面板（交叉点详情/论文/学者/启动辩论）**或** 发现结果面板（DiscoveryPanel）
  - 发现模式: 展示研究问题 → 相关学科（含相关度百分比）→ 推荐组合（含研究方向/空白标记/操作按钮）

### 3. 辩论页

- 入口页 `/debate`: 学科选择 + 模式选择（自由讨论/正反辩论）+ AI 推荐 + Agent 预览 + 历史列表
- 会话页 `/debate/:id`:
  - 左侧: Agent 阵容侧边栏（姓名 + 学科 + 性格标签 + 立场）
  - 中央: 消息流（按轮次分组，Agent 头像/颜色区分）
  - 底部: 下一轮 / 结束辩论 → 四段式总结（共识/分歧/开放问题/建议方向）
  - 辩论完成后: "生成论文大纲" 按钮 → 输入研究方向 → 跳转 PaperEditor
  - 已有论文草稿列表（可直接跳转）
- Agent 阵容: 见下方 "Agent 阵容设计" 章节
- 正反辩论模式: Advocate(正方) / Challenger(反方) 交替 + 中立 Moderator

## Agent 阵容设计

> 核心理念：模拟真实学术团队的层级结构和个性多样性。

### 三个独立维度

Agent 的身份由三个**互相独立**的维度构成：

| 维度 | 属性 | 取值 | 决定方式 |
|------|------|------|----------|
| 学科 | `discipline_id` | 用户选的学科 | 用户选择 |
| 职级 | `rank` | professor / associate / assistant | 由学科权重决定 |
| 性格 | `persona` | pioneer / rigorous / pragmatic / skeptic | 每次创建时随机分配 |

### 设计决策及其原因

**为什么性格随机分配，不绑定学科？**

把"数学=严谨"、"心理学=开拓"固化下来是学科刻板印象。现实中数学家也做大胆猜想，心理学家也做严谨实验。如果 AI 先入为主给学科贴思维标签，反而限制了讨论的可能性。性格是辩论中的"角色需要"（需要有人提新想法、有人质疑），不是学科的固有属性。同一个学科组合每次发起辩论，性格分配可能完全不同，讨论也会有不同的走向。

**为什么每学科 2 个 Agent（教授+副教授）而不是 1 个？**

真实学术团队中，同一个方向内部就有主次和分歧。教授有最终决策权但助理教授可能有更前沿的视角。这种"内部张力"让讨论更丰富——不只是学科间碰撞，学科内部也有对话。

**为什么学科权重可以由用户指定？**

研究者对自己要做的课题有主次判断。比如"AI + 心理学"做人机交互，两者权重接近；但做"深度学习优化"，AI 权重明显高于心理学。让用户设定核心/辅助方向，系统据此决定团队大小（核心方向 2 人，辅助方向 1 人），比全部让 AI 判断更尊重研究者的意图。

**为什么不设置则 LLM 自动判断，而不是平均分配？**

平均分配虽然公平但不真实。LLM 可以根据学科组合和辩论主题推断哪个学科更核心。如果 LLM 也失败了，才 fallback 到平均分配。

### 权重决定链

```
用户手动指定 discipline_weights
       ↓ (未指定的学科)
LLM 根据学科组合 + 辩论主题推断 (0-100)
       ↓ (LLM 失败)
平均分配 (weight = 50)
       ↓
权重 >= 40 → 2 人 (Professor + Associate/Assistant Professor)
权重 < 40  → 1 人 (Professor only)
总人数上限 8 (不含 Moderator)
```

### 发言规则

| 维度 | Professor | Associate/Assistant |
|------|-----------|---------------------|
| 发言长度 | 300-500 words | 200-350 words |
| max_tokens | 1500 | 1000 |
| 第 1 轮发言顺序 | 先发言 | 后发言 |
| 后续轮次 | 交错发言 (Prof A → Prof B → Assoc A → Assoc B) | 交错发言 |
| 总结时 | Moderator 更多引用 | 补充视角 |

### System Prompt 结构

每个 Agent 的 system prompt 包含：
1. 学科身份 + 职级（"You are a Professor/Associate Professor specializing in X"）
2. 权重暗示（"核心方向" vs "辅助视角"）
3. 队友关系（教授可以指导同学科的副教授，副教授可以补充细节或提出不同看法）
4. 性格描述（随机分配的 persona）
5. 立场（仅 debate 模式：advocate/challenger）
6. 发言规则（字数限制、引用要求、互动要求）

### Future: Agent 职能维度 (role)

> 当前的 persona 是性格维度，但缺少**工作方式**维度。
> 真实学术团队中，理论组和实验组是**分工**，不是性格差异。
> "实验"不是一种性格，是一种工作方式。
>
> 下一步应引入独立的 `role` 属性 (theorist / experimentalist)，与 persona 正交：
>
> | | 开拓型 | 严谨型 | 实用型 | 批判型 |
> |---|---|---|---|---|
> | **理论研究员** | 提新理论 | 推导证明 | 理论→应用 | 质疑假设 |
> | **实验研究员** | 设计新实验 | 统计验证 | 实验→产品 | 挑方法漏洞 |
>
> 每个 Agent 同时拥有 学科 x 职能 x 性格 三个独立维度。
>
> **重要澄清：experimentalist Agent 不做实验，只做实验方案设计建议。**
> 真正的实验由论坛社区中有资源的研究者承接（见"产品哲学：AI 做理论，社区做实验"）。
> experimentalist Agent 的价值是帮接手实验的人设计更好的方案：
> - 推荐实验范式（A/B 测试 vs 纵向追踪 vs 案例研究）
> - 统计功效分析和样本量建议
> - 跨学科实验设计的混杂变量提醒
> - 引用类似实验设计的参考论文
>
> **待层级化稳定后单独实施。**

### 5. 论文编辑页 (`/paper/:draftId`)

## 图谱边的数据来源与语义

画布上的连线（edge）有三种来源，视觉样式不同：

| 类型 | status | 数据来源 | 视觉 | 含义 |
|------|--------|----------|------|------|
| 深度分析 | `active` | Intersection 表（手工策展） | 紫色实线 | 已有深入研究的交叉关系 |
| 论文证据 | `evidence` | paper_discipline 共同论文计数 | 灰色实线（粗细随论文数） | 有论文同时涉及这两个学科 |
| 研究空白 | `gap` | Intersection 表 status=gap | 红色虚线 | 标记为待研究的交叉 |

**关键设计决策：Intersection 超集不投影**

如果数据库中有一个三元交叉点 [A, B, C]，而用户只选了 [A, B]，图上**不会**出现这条边。只有当用户至少选了 A、B、C 三个时，这个交叉点才会显示。原因：把三元关系投影到二元边上会造成语义错误——用户会以为"A 和 B 之间存在这个研究"，但实际上那个研究需要 C 的参与。

如果未来需要展示"部分相关"的交叉点，应添加新的 edge status（如 `partial`），用不同的视觉样式标注"此交叉点还涉及你未选中的学科"。

**Evidence 边不可点击**

只有 `intersection_id` 不为 null 的边（即 active/gap）才响应点击跳转到详情。Evidence 边只有 paper count 信息，没有策展内容，所以光标显示为 default 而非 pointer。

### 社区流（论坛的两大核心功能）

论坛不只是讨论区。它承担两个不可替代的功能：

**功能一：专家验证研究方向**

AI Agent 能发现空白、能论证理论，但 AI 不是专家。一个真正在这个交叉领域工作了十年的人，看一眼就知道这个方向靠不靠谱、有没有人已经在做、有什么 AI 不知道的坑。论坛让专家来做这个判断——确认方向、修正方向、或者否定方向。这比 AI 自说自话可靠得多。

**功能二：实验验证与协作**

理论再漂亮，没有实验数据就是空谈。世界各地的实验室有不同的资源优势——有人有 fMRI，有人有十万用户的行为数据，有人有特定人群的被试池。论坛让方向和资源自然匹配，让有能力做实验的人找到值得做的方向，让有好方向的人找到能做实验的人。

**这两个功能合在一起，让 Agent X Lab 真正为所有人类的科研做贡献：AI 打破信息壁垒发现方向，专家把关质量，社区协作完成验证，成果回馈到公共知识图谱，全世界的研究者都能受益。**

```
辩论完成
  → 自动生成论坛帖（标注阶段："理论论证完成，寻求实验验证"）
  → 帖子包含：辩论四段式总结 + 论文草稿链接 + 涉及学科标签
  → 按学科分区推送，相关领域的研究者能看到
  ↓
社区互动
  → 有实验资源的用户认领方向（"我的实验室可以做这个"）
  → 讨论实验设计方案 → 其他用户/AI Agent 提供方法论建议
  → 实验进展更新帖
  ↓
成果反馈
  → 实验结果贴回论坛 → 关联到原始辩论和交叉点
  → 知识图谱更新：虚线（空白）→ 实线（已有证据）
  → 新论文入库 → 下一轮 AI 辩论有了更扎实的证据
  ↓
正循环
  → 更多空白被发现 → 更多理论被论证 → 更多实验被做出
  → 知识图谱越来越厚 → AI Agent 越来越聪明 → 社区越来越活跃
```

**论坛帖子的生命周期：**

| 阶段 | 状态标签 | 含义 |
|------|----------|------|
| AI 辩论完成 | `理论就绪` | 跨学科方向已论证，等待实验验证 |
| 有人认领 | `实验进行中` | 某研究者/实验室接手，正在设计或执行实验 |
| 实验完成 | `已验证` / `已证伪` | 实验结果出炉，反馈到知识图谱 |
| 衍生讨论 | `新问题` | 实验结果引发新的理论问题，可能触发新一轮辩论 |

**积分系统围绕贡献而非活跃度：**

| 行为 | 积分 | 理由 |
|------|------|------|
| 发现新的研究空白 | 中 | 前半段贡献 |
| 完成 Agent 辩论并生成论文 | 中 | 前半段贡献 |
| 认领实验方向 | 高 | 后半段关键动作 |
| 提交实验结果 | 最高 | 完成闭环 |
| 提供方法论建议 | 中 | 降低实验门槛 |
| 高质量讨论（被引用） | 低-中 | 知识沉淀 |

## 核心实验：激发 AI 的想象力

> Agent X Lab 本身是一个巨型实验。实验目标：**AI 在什么条件下能产生人类没想到的、有价值的跨领域联想？**

### 为什么这个实验有意义

AI 没有真正的想象力，但它有一个独特能力：它同时"读过"所有领域的文献。一个人类心理学家可能一辈子不会读量子物理的论文，但 AI 的训练数据里两者都有。当两个领域放在一起辩论时，AI 可能从训练数据的深层统计关联中找到人类因为学科壁垒根本不会想到的连接。

这种"连接"不是逻辑推理出来的，是从海量数据中涌现出来的。跨学科辩论是激发这种涌现的手段，产出的"跨领域火花"是实验数据，社区专家验证是 ground truth。

### Agent 持久记忆：三层认知架构

每个 Agent 的持久身份由 `discipline_id + rank` 决定，跨辩论共享记忆（Zep user_id: `agent-{discipline_id}-{rank}`）。

认知内核分三层，每层有不同的写入和淘汰规则：

**第一层：事实与证据**

- 示例："Transformer 架构在 NLP 中取得突破"
- 客观、可验证的知识
- 写入标准：有出处可验证
- 淘汰标准：被新证据推翻即更新

**第二层：论证与立场**

- 示例："注意力机制可以类比为社会心理学中的选择性注意"
- 有逻辑推导链的观点
- 写入标准：有完整论证链路
- 淘汰标准：被更强论证取代（但原文和反方论证都保留）
- 不以共识为标准——爱因斯坦反对量子力学的 EPR 论证是少数派，但论证严密，这类观点必须保留

**第三层：跨领域火花**

- 示例："如果把社会网络中的信息传播模型反过来用在神经网络的剪枝上呢？"
- 可能没有严密论证，可能看起来疯狂
- 写入标准：只要有启发性就留
- 淘汰标准：**永不自动淘汰**，只标注后续验证状态
- 这是平台的灵魂。今天疯狂的类比，五年后可能变成一个新领域

### 为什么不用共识作为认知更新的标准

科学史上最重要的突破几乎都是一个人对抗共识的结果。如果"共识才能更新认知"，大陆漂移理论、幽门螺杆菌致病假说、EPR 悖论全部会被过滤掉。

一个观点值不值得保留，不看多少人同意，看：

- 论证链路是否完整
- 是否引用了可验证的证据
- 是否回应了已知的反驳
- 是否具有启发性（第三层的标准更宽——启发性本身就是价值）

### 防退化机制：不靠共识，靠分层规则

核心担忧：低质量辩论多了，Agent 记忆被污染，越来越蠢。

解决方案：讨论记录（大库）和认知内核（小库）是两套系统。

- **讨论记录**：所有辩论都完整保存，只追加不删除，任何人可查阅。像公共图书馆。
- **认知内核**：只有通过门槛的认知才写入。像 Agent 的大脑。

认知更新不是每场辩论自动触发，而是通过"认知蒸馏"——LLM 对比 Agent 已有认知和本次辩论新信息，输出合并后的认知（不是追加，是重写）。事实层和论证层有容量上限，超出时压缩最旧条目；但火花层只压缩表述，不删条目。

覆盖已有认知的门槛高于写入新认知的门槛，就像学术界发表新观点容易但推翻已有理论很难。

### 跨领域火花采集与测量

辩论过程中，每个 Agent 发言后自动评估是否包含跨领域类比或意外联想，提取为 `Spark` 记录：

- 火花内容、来源学科、目标学科
- 新颖度类型（类比 / 概念迁移 / 融合 / 反转）
- 新颖度评分（0-1，综合学科距离、概念新组合度、与已有火花的差异度）

每场辩论的实验条件（学科数量、学科距离、Agent 数量、性格分布、轮次、权重分布）全部记录，用于后续分析"什么条件下火花更多/质量更高"。

### 火花进入社区

火花在论坛中是独立的帖子类型，带有：新颖度评分、涉及学科标签、来源辩论链接、社区投票（赞同/质疑/想做实验）、验证状态徽章。

验证状态生命周期：`未验证` → `有人感兴趣` → `实验中` → `已验证` / `已证伪但启发了新方向`

社区验证结果回写到 Agent 的火花层记忆，形成正循环。

### 这个实验最终要回答的问题

1. AI 什么时候会产生真正新颖的跨领域联想（而不是重复训练数据里已有的跨学科论文）？
2. 什么条件下产生得更多、质量更高？（学科距离？辩论轮次？Agent 数量？性格组合？教授 vs 副教授？）
3. 这些联想中，有多少最终被人类专家认为有价值？
4. Agent 的记忆积累是否会让它在后续辩论中产生更高质量的火花？（学习曲线）
5. 不同性格的 Agent 产生火花的模式是否不同？（开拓型是否真的更有想象力，还是批判型反而因为"反向思考"产生了更多意外联想？）
